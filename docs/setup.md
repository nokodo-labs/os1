# setup

development and deployment guide for nokodo.

## prerequisites

- **python 3.13+** with [uv](https://docs.astral.sh/uv/)
- **node.js 24+** with [pnpm](https://pnpm.io/)
- **docker** with docker compose
- **postgres 17** (provided via docker)

## quick start

```bash
cd .docker && docker compose --profile deps up -d
```

this boots postgres locally so you can run backend/frontend from your IDE with hot reload.

## local development

### backend

```bash
cd backend
uv sync --all-extras
cp .env.example .env
uv run uvicorn api.main:app --reload
```

> migrations run automatically on startup.

### frontend

```bash
cd frontend
pnpm install
pnpm run dev
```

### console (admin)

```bash
cd console
pnpm install
pnpm run dev -- --host --port 8383
```

### generating API types

after backend API changes, regenerate the TypeScript client using VS Code tasks:

- **Frontend: Generate API Types** - runs `pnpm run generate:api-types`
- **Console: Generate API** - runs `pnpm run generate:api`

> requires backend running at `http://localhost:1383`

## VS Code setup

the project includes comprehensive VS Code integration.

**tasks** (`Ctrl+Shift+P` → "Tasks: Run Task"):

- **Backend**: Install Dependencies, Run Server, Run Tests, Type Check
- **Frontend**: Install Dependencies, Dev Server, Build, Generate API Types
- **Console**: Install Dependencies, Dev Server, Build, Generate API Types
- **Docker**: Ensure Dev Deps, Build All, Up, Down

**debug configurations** (press `F5`):

- **Python: FastAPI** - debug backend server
- **Python: Pytest** - debug current test file
- **Full Stack** (compound) - runs backend + frontend together

### quick start

1. open workspace in VS Code
2. accept prompt to install recommended extensions
3. press `F5` → choose a debug configuration

## docker commands

```bash
cd .docker

# start/stop
docker compose --profile deps up -d        # dependencies only
docker compose --profile local up -d       # full local stack
docker compose down                        # stop services
docker compose down -v                     # stop and delete volumes

# logs
docker compose logs -f [service]

# rebuild
docker compose --profile local up -d --build
```

## database migrations

> migrations run automatically on backend startup. manual commands are rarely needed.

### creating a new migration

use the `backend: autogen migration` VS Code task, or run the tool directly from the repo root:

```bash
uv run --project backend python tools/autogen-migration.py --message "description"
```

this creates a temporary isolated database, applies all existing migrations to it, runs `alembic revision --autogenerate` against the diff, then saves the result to `temp/alembic-autogen/`. review and edit the generated file, then move it into `backend/api/migrations/versions/`.

options:

- `--message "description"` - revision message (default: "autogenerated migration")
- `--output path/to/file.py` - custom output path
- `--upgrade-revision <rev>` - revision to apply before autogenerate (default: head)
- `--keep-database` - do not drop the temp database after running
- `--check` - check for drift only; doesn't require a message

### other alembic commands

```bash
cd backend

# apply pending migrations
uv run alembic -c api/migrations/alembic.ini upgrade head

# roll back the last migration
uv run alembic -c api/migrations/alembic.ini downgrade -1

# show migration history
uv run alembic -c api/migrations/alembic.ini history
```

## production deployment

### using pre-built images

CI/CD automatically builds and publishes docker images to GitHub Container Registry on every commit.

**image tags:**

- `ghcr.io/nokodo-labs/nokodo-ai:latest` - latest release
- `ghcr.io/nokodo-labs/nokodo-ai:dev` - latest dev branch
- `ghcr.io/nokodo-labs/nokodo-ai:v1.2.3` - specific version

**deploy:**

```bash
cd .docker
docker compose pull
docker compose up -d
```

### environment configuration

**backend/.env** (required for production):

```
DATABASE_URL=postgresql+psycopg://user:pass@db:5432/nokodo-production
SECRET_KEY=generate-secure-random-key-here
DEBUG=False
APP_ENV=production
CORS_ORIGINS='["https://yourdomain.com"]'
```

### deployment steps

1. configure environment - update `.env` files and `.docker/docker-compose.yml`
2. pull images - `docker compose pull`
3. start services - `docker compose up -d`
4. verify - `docker compose logs -f`

## environment variables

### backend/.env

```
DATABASE_URL=postgresql+psycopg://nokodo-ai-admin:nokodo-ai@127.0.0.1:5432/nokodo-ai-dev
SECRET_KEY=change-in-production
DEBUG=True
```

## troubleshooting

**port conflicts** - change ports in `.docker/docker-compose.yml`

**DB connection failed** - `docker compose restart db`

**types out of sync** - regenerate using the VS Code tasks after backend changes
